    Springboot+vue框架部署

    2023-12-12

    Spring Boot是一个用于构建Java应用程序的开发框架，它提供了快速开发、简化配置和自动化任务的特性。Vue.js是一个流行的JavaScript前端框架，用于构建交互式的用户界面。

    Spring Boot + Vue框架的优点：

    1. 分离前后端开发：Spring Boot提供了强大的后端支持，而Vue.js则专注于构建交互式的前端界面，使得前后端开发可以相对独立进行。
    2. 灵活性：Spring Boot的自动化配置和Vue.js的组件化开发使得系统更加灵活，易于维护和扩展。
    3. 性能：Spring Boot具有快速启动和高性能的特点，而Vue.js采用虚拟DOM技术，提高了页面渲染效率，有利于提升系统的整体性能。
    4. 生态系统：Spring Boot和Vue.js都拥有庞大的社区和丰富的生态系统，提供了大量的插件和组件，可以加速开发过程。

    Spring Boot + Vue框架的缺点：

    1. 学习曲线：对于新手来说，学习并掌握Spring Boot和Vue.js可能需要一定的时间和精力。
    2. 复杂性：集成前后端框架需要一定的技术能力和经验，对开发人员的要求较高。

    功能性：

    1. 全栈开发：结合Spring Boot和Vue框架可以实现前后端分离的全栈开发，提高开发效率和灵活性。
    2. RESTful API：Spring Boot提供了便捷的RESTful API开发支持，与Vue的前端请求配合良好，实现数据的交互和展示。
    3. 响应式页面：Vue框架可以实现响应式页面，使得前端页面的交互性和用户体验更加出色。

    总的来说，Spring Boot + Vue框架结合后能够提供一个高效、灵活和性能优越的全栈开发解决方案，但也需要开发人员具备一定的技术水平和经验。

    部署

    在部署Spring Boot和Vue框架时，可能会用到以下依赖和构建工具：

    依赖和构建工具

    1. Maven/Gradle：用于构建和管理Spring Boot后端应用程序的依赖关系，以及打包成可执行的JAR文件。
    2. Node.js：Vue框架依赖Node.js环境，在本地开发和构建Vue应用程序时需要安装Node.js。
    3. Vue CLI：Vue的官方构建工具，用于创建、构建和管理Vue应用程序，可以通过Vue CLI创建Vue项目，并利用其构建功能生成静态资源文件。
    4. Npm/Yarn：用于安装Vue项目中的依赖包，以及执行构建、打包等前端项目管理任务。
    5. Webpack：Vue CLI使用Webpack作为默认的打包工具，用于构建和打包Vue应用程序的静态资源。
    6. HTTP服务器（如Nginx、Apache等）：用于部署和提供Vue应用程序的静态资源，以及与后端应用程序进行通信。
    7. Java运行时环境（JRE）：在部署Spring Boot应用程序时，需要在目标服务器上安装适当版本的Java运行时环境。

    这些依赖和构建工具在整个部署过程中发挥着重要作用，它们可以帮助开发人员管理项目依赖、构建应用程序，并最终将应用程序部署到生产环境中。

    本次部署过程是后端:Springboot负责前端控制器之间通信，前端:Vue页面渲染和element组件实现快速创建组件以及美化

    1. 简化CRUD操作：MyBatis Plus封装了通用的CRUD操作，简化了数据库操作的编写。
    2. 代码生成器：MyBatis Plus提供了代码生成器，可以根据数据库表自动生成实体类、Mapper接口和XML映射文件，减少了开发人员的重复劳动。
    3. Lambda查询：MyBatis Plus支持使用Lambda表达式进行条件构造，提高了查询条件的可读性和灵活性。
    4. 分页插件：MyBatis Plus提供了强大的分页插件，可以轻松实现分页查询功能。
    5. 通用方法：MyBatis Plus提供了许多通用的方法，如批量插入、批量更新等，减少了开发人员的开发工作。
    6. 逻辑删除：MyBatis Plus支持逻辑删除功能，可以方便地处理数据的逻辑删除操作。

    总的来说，MyBatis Plus是MyBatis的一个增强工具，它简化了数据库操作的编写，提高了开发效率，并提供了许多实用的功能和增强特性，使得基于MyBatis的开发更加便捷和高效。

    CodeGenerator

    CodeGenerator（代码生成器）通常是指一种工具或框架，用于根据预定义的模板和规则自动生成代码文件，以加速软件开发过程。代码生成器可以生成数据库访问层、业务逻辑层、API接口等代码，从而减少开发人员的重复性工作，提高开发效率。

    一般情况下，代码生成器具有以下一些功能和特点：

    1. 模板定义：代码生成器通常会定义一些代码模板，用于生成特定编程语言的代码文件，这些模板可以包含预定义的结构、代码片段和规范。
    2. 配置规则：开发人员可以通过配置文件或界面设定生成代码的规则，如指定数据表和字段对应的代码文件、命名规范、注释格式等。
    3. 自动化生成：在配置好规则和模板后，代码生成器可以自动根据规则读取数据库结构，并生成对应的代码文件。
    4. 灵活性：一些代码生成器提供了一定的灵活性，允许开发人员根据需要定制模板和规则，以满足特定的开发需求。
    5. 集成性：代码生成器通常能够与主流的开发框架和工具集成，如与Spring框架、MyBatis、Hibernate等集成，以便生成符合特定框架标准的代码。

    总的来说，代码生成器是一种工具，可以根据预定义的规则和模板，自动生成特定编程语言的代码文件，从而提高开发效率，减少重复性工作，同时也有利于保持代码的一致性和规范性。

    Maven

    是一个用于Java项目管理和构建的强大工具。它提供了一种标准化的方式来构建项目、管理项目依赖、运行单元测试、生成文档等。Maven使用基于项目对象模型（POM）的概念，通过XML文件来描述项目的结构和依赖关系。

    Maven优点

    1. 标准化项目结构：Maven鼓励使用标准的项目结构，使得项目更易于维护、扩展和传递。
    2. 依赖管理：Maven能够自动下载并管理项目所需的第三方依赖，简化了项目配置和构建过程。
    3. 插件生态系统：Maven具有丰富的插件生态系统，可以扩展构建过程，实现自定义的操作和功能。
    4. 生命周期管理：Maven定义了一套清晰的构建生命周期，包括编译、测试、打包、部署等阶段，使得构建过程更加可控。
    5. 多模块支持：Maven能够轻松管理多模块项目，定义父子模块的依赖关系和继承关系。
    6. 中央仓库：Maven中央仓库作为一个公共的资源库，能够提供大量的开源库供用户使用。

    Maven缺点

    1. 学习曲线：对于新手来说，Maven的概念和配置可能需要一定的学习成本。
    2. 约定大于配置：Maven强制使用一些约定来标准化项目结构，有时候可能会限制一些自定义需求。
    3. 性能：在下载依赖时，Maven可能会受到网络速度的影响，导致构建速度较慢。

    总的来说，Maven作为一个标准化的项目管理和构建工具，具有丰富的功能和插件生态，能够帮助开发人员管理和构建Java项目，但也需要开发人员花一定的时间学习和理解其概念和使用方法。

/**

       // pc.setModuleName("admin"); //模块名

        pc.setMapper("dao");

        strategy.setInclude("commentary");//生成指定的表
        strategy.setNaming(NamingStrategy.underline_to_camel);//数据库表映射到实体的命名策略

        strategy.setColumnNaming(NamingStrategy.underline_to_camel);//数据库表字段映射到实体的命名策略
        strategy.setEntityLombokModel(true); // lombok 模型 @Accessors(chain = true) setter链式操作
        strategy.setRestControllerStyle(true); //restful api风格控制器
        strategy.setControllerMappingHyphenStyle(true); //url中驼峰转连字符
        strategy.setVersionFieldName("version");//设置乐观锁
        strategy.setLogicDeleteFieldName("deleted");
        mpg.setStrategy(strategy);


        // 6、执行
        mpg.execute();
    }
}

    Springboot-vue的基础传值

    众所周知，后端和前端必须相互传值才能完成一系列的数据交互以实现一些各种各样的功能。

    前端大部分时候只负责渲染页面和展示数据，而后端则包揽了数据的计算和获取，把最终结果交给前端。

    我们所使用的框架基本都是基于后端服务的，比如本项目中的mybatis框架，mybatis plus和maven包括CodeGenerator等，都是帮助后端快速创建项目结构和减轻重复性代码。

    在后端中需要以性能为主，而前端需要保证页面的美观度和可读性。

    其中最主要的就是相互传值

    前端想要传值给后端需要以路径访问的形式访问，后端就可以接收到，在Spring中，前端访问后端定义好的路径后只需要在后面添加

    ?id=1

    就能将值传递给后端，然后后端要在方法中填写对应的参数就可以，例如：

    前端

    axios.get('comment/list?id=' + id)

    后端

    @GetMapping("/list")
    public List<Comment> findlistByID(@RequestParam String id){
        Comment comment = commentMapper.selectById(id);
        List<Comment> resultList = Collections.singletonList(comment);
        return  resultList;
    }

    使用@RequestParam就可以获取到

    tip:即使是id，在传值过程中也会变成String类型，如果需要int类型那么使用JAVA方法转换类型即可

    int num = Integer.parseInt(str);

    前端通过访问路径来获取后端的数据，在使用了Spring+mybatis+mybatis plus+CodeGenerator后，大部分操作只需要在前端控制器编写最核心代码。

    以下示例:

    package com.example.springbootvue.controller;
import com.baomidou.mybatisplus.core.conditions.query.QueryWrapper;
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;
import com.example.springbootvue.dao.ChartsMapper;
import com.example.springbootvue.entity.Charts;
import com.example.springbootvue.service.ChartsService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import java.util.List;
/**
 * <p>
 *  前端控制器
 * </p>
 *
 * @author frr
 * @since 2023-12-11
 */
@RestController
@RequestMapping("/charts")
public class ChartsController {
    @Autowired
    private ChartsService chartsService;
    @Autowired
    private ChartsMapper chartsMapper;

    @PostMapping("list")
    public Page<Charts> findlist(@RequestBody Page<Charts> page){
        Page<Charts> p = chartsService.page(page);
        System.out.println(p);
        return p;
    }

    @GetMapping("/mlist")
    public List<Charts> mlist(@RequestParam String input){
        QueryWrapper<Charts> fuzz = new QueryWrapper<>();
        fuzz.like("keywords",input);
        List<Charts> charts = chartsMapper.selectList(fuzz);
        System.out.println(charts);
        return charts;
    }
}

    在这个案例中，使用了两种方式传参和接收数据库数据

    模糊查询和分页查询

    首先来说明模糊查询

    模糊查询

    QueryWrapper<Charts> fuzz = new QueryWrapper<>(); //创建模糊查询语句 然后<填写自己需要查询的表的实体类>
fuzz.like("keywords",input); //参数为 查询的表的关键字 和 匹配的值 此处为动态数据传参的参数
List<Charts> charts = chartsMapper.selectList(fuzz); //将查询语句添加进查询表里，返回值为 List 就已经拿到了数据

    分页查询

    @PostMapping("list")
    public Page<Charts> findlist(@RequestBody Page<Charts> page){
        Page<Charts> p = chartsService.page(page);
        System.out.println(p);
        return p;
    }

    分页查询的难点在于参数是Page并且返回值也是Page

    前端需要在Data里创建对象来作为分页查询的参数

     data: {
    page:{
        current:1,
        Size:3,
        total:0
    }
}
    • current: 代表当前页数，即需要查询的页码。
    • Size: 代表每页显示的记录数，即每页需要展示的数据条数。
    • total: 代表总记录数，即数据总条数。

    然后就可以作为参数添加到方法里随时调用

    findList(){
              // debugger
              axios.post('charts/list',page=this.page).then(res=>{
                  // console.log(res)
                  this.tableData = res.data.records;
                  console.log(this.tableData);
                  this.page.total = res.data.total;
              })
         	}

    只需要修改page对象的值再次发送请求就可以获取到不同页的数据了
